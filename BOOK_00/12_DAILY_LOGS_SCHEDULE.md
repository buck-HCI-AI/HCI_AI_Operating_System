## The Tie-In: Daily Logs Are Schedule Intelligence Inputs

Daily logs are not just records. They are the ground truth signal for schedule analysis. Every log entry must be evaluated against the planned schedule to detect variance, flag risk, and generate alerts.

```
DAILY LOG SUBMITTED (WF-SUPER)
    │
    ▼
SCHEDULE ANALYSIS
    │   Compare actual progress to scheduled activities
    │   Detect: manpower shortage, weather impact, material delay,
    │           inspection hold, constraint blocker
    ▼
VARIANCE DETECTION
    │   Is actual progress behind planned by >10%?
    │   Did a constraint push an activity?
    │   Did an inspection fail?
    ▼
STATUS-CHANGE REPORT (if variance detected)
    │   Affected activity, baseline, current, variance,
    │   cause, responsible party, risk level, recovery, decision needed
    ▼
RISK TABLE UPDATE (if risk-level = high or critical)
    │
    ▼
PROJECT BRAIN CACHE INVALIDATED
    │
    ▼
MORNING BRIEF ALERT (next day)
```

## Schedule Variance Table (Phase 9.2)

```sql
CREATE TABLE schedule_variance (
    id                 SERIAL PRIMARY KEY,
    project_id         INTEGER REFERENCES projects(id),
    daily_log_id       INTEGER REFERENCES daily_logs(id),
    activity_name      TEXT,
    baseline_status    TEXT,
    current_status     TEXT,
    variance_days      INTEGER,
    variance_pct       NUMERIC,
    cause              TEXT,
    responsible_party  TEXT,
    risk_level         TEXT,   -- 'low','medium','high','critical'
    recovery_action    TEXT,
    decision_needed    TEXT,
    recommended_notification TEXT,
    detected_at        TIMESTAMPTZ DEFAULT now()
);
```

## Schedule Analysis Logic

```python
# schedule_intelligence_svc.analyze_log(daily_log_id)

def analyze_log(daily_log_id: int) -> dict:
    log = pg_one("SELECT * FROM daily_logs WHERE id = %s", (daily_log_id,))
    
    # Claude analyzes the log against any known schedule context
    prompt = f"""
    Daily log for {log['project_id']} on {log['log_date']}:
    Work performed: {log['work_performed']}
    Constraints: {log['constraints']}
    Field risks: {log['field_risks']}
    Subcontractor progress: {log['subcontractor_progress']}
    
    Based on this log, identify any schedule variance or risk.
    Return: affected_activity, baseline_status, current_status, 
            variance_estimate, cause, risk_level, recovery_action.
    """
    
    analysis = ask_claude(prompt, system=SCHEDULE_ANALYSIS_SYSTEM)
    
    if analysis['risk_level'] in ('high', 'critical'):
        write_to_risks_table(analysis)
    
    write_to_schedule_variance(analysis, daily_log_id)
    return analysis
```
